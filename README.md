# TodoList
### 출처 : 기초에서 수익 창출까지, 안드로이드 프로그래밍 A-Z

## 사전지식 
1. Room Lib  
    : 로컬 데이터 베이스  
    : SQLite(since API1) -> Room ( 추상화 계층 추가)  
    : Entity,  Data access object,  Database로 구성
     
<br>

2. RecyclerView  
  : 효율적으로 작동하는 리스트 생성  
  : RecyclerView는 기존의 뷰의 내용만 바꾸고 재활용

4. Annotation
    - 자바(코틀린) 코드에 추가할 수 있는 메타데이터의 한 형태  
    - 보일러 플레이트 코드를 줄여 간결한 코드 작성 가능
    - Room, Retrofit 라이브러리에서 자주 사용
  
<br>

5. Boilerplate code
    - 최소한의 변경으로 여러곳에서 재사용되며, 반복적으로 비슷한 형태를 띄는 코드     

<br>

6. Singleton Pattern  
    - 소프트웨어 디자인 패턴 중 하나
    - 프로그램 내에서 객체 하나를 공유하는 패턴

<br>

7. Companion Object
   - 어떤 클래스의 모든 인스턴스가 공유하는 객체를 만들고 싶을 때 사용
   - 클래스당 한 개만 가질 수 있다.
<br><br><br>
